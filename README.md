# IT Audit User Access Review Dashboard

## 📌 Project Overview
This project simulates an IT Audit review of user access controls using Power BI. The objective was to identify access control weaknesses, assess risk levels, and provide actionable remediation recommendations.

The dashboard focuses on key access control issues such as dormant accounts, unused accounts, and delayed access removal for terminated employees.

---

## 🎯 Audit Objective
To evaluate whether user access is:
- Appropriately provisioned
- Regularly reviewed
- Timely removed upon termination

---

## 🛠️ Tools Used
- Power BI 
- DAX 
- Excel (sample dataset) 

---

## 🔍 Key Risk Areas Analyzed
- Dormant Accounts 
- Never Logged In Accounts 
- Post-Termination Access 
- Terminated but Active Accounts 
- High-Risk Access by System and Access Level 

---

## 📊 Risk Scoring Methodology
Risk scores were calculated using a standardized approach:

Risk Score = Impact × Likelihood (5×5 matrix) 

Risk levels were categorized as:
- Low 
- Medium 
- High 
- Critical 

This approach ensures consistent prioritization of access control issues.

---

## 📈 Dashboard Pages

### 1. Executive Risk Overview
- Summary of total exceptions by risk level 
- High-level insights into access control risks 

### 2. Detailed Findings & Audit Evidence
- Detailed list of access control exceptions 
- Audit results (Pass/Fail) 
- Control effectiveness status 
- Supporting evidence fields (hire date, termination date, last login)

### 3. Risk Scoring Methodology
- Explanation of risk calculation approach 
- Definitions of key access control issues 

### 4. Access Control Risk Analysis & Key Drivers
- Identification of primary drivers of high-risk exposure 
- Breakdown by:
  - Access Level 
  - Control Failure Type 
  - System 

---

## 🚨 Key Findings
- Dormant accounts represented the highest volume of access control issues 
- Termination-related access posed the highest risk due to potential unauthorized activity 
- High-risk access violations were heavily concentrated within the ERP system 
- Standard user accounts accounted for the majority of high-risk exposure 

---

## ✅ Recommendations
- Remove access for terminated users immediately 
- Strengthen deprovisioning processes 
- Implement periodic user access reviews 
- Disable inactive and unused accounts 
- Enhance monitoring for high-risk systems such as ERP
